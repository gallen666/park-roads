第一章 · 设计哲学:为什么是「诊疗体系」

1.1 园区数字化转型的三大客户痛点

过去十年,园区数字化转型咨询服务存在三个长期未被解决的客户体验痛点。这三大痛点不仅造成客户决策成本极高,也使得咨询机构难以规模化复制其交付能力。

  1. 不知道自己处在哪个阶段。客户面对「数字化转型」这一宏大命题时,无法判断自身当前所处的成熟度水平、与同类园区差距以及优先级排序。

  2. 不知道下一步应做什么。即便了解自身现状,客户也难以在十几个候选议题(智慧楼宇、能源管理、招商系统、双碳监测、人才公寓数字化……)中确定下一笔预算的最优投放方向。

  3. 不知道为什么要为咨询付钱。咨询服务的「软性交付物」难以被园区运营方、上级国资委或财政评估,导致咨询费议价能力薄弱,也使得后续追加服务的转化率极低。

Park-ROADS 团队总结:园区数字化转型咨询的本质难题,是「认知失序」+「服务无标准」+「价值无凭证」。要解决这一难题,需要一个让客户「秒懂、敢付、愿意续费」的服务体系。

1.2 医院五步法的内在逻辑

现代医院体系是人类发明的最成熟的「按阶段付费的专业服务体系」。从挂号、体检、诊断、治疗到复诊,五个环节构成一个高度标准化、可循环、可分级、可计费的服务链路。

  • 标准化:每一步都有清晰的输入、输出、SOP 与文档;

  • 可循环:复诊环节将周期性疗效追踪嵌入体系,使一次性服务演化为持续关系;

  • 可分级:挂号导诊机制将不同复杂度的患者分配到普通门诊、专家门诊、综合会诊;

  • 可计费:每个环节都有清晰的收费项目、保险结算编码与质控指标,患者「为什么付钱」一目了然;

  • 可监管:从挂号编号到医保结算、从病历归档到 DRG 分组,全链路可审计、可追溯。

医院五步法的成功之处不在于「医术高超」,而在于「服务体系标准化」。这一逻辑可以被几乎完整地移植到园区数字化转型咨询。

2.2 五系统之间的数据贯通

五系统不仅是流程上的衔接,更是数据贯通的整体。每个系统都向下游系统提供结构化数据,使客户「越用越懂、越用越省」。

第三章 · 园区数字化咨询挂号系统

3.1 功能定位

挂号系统是客户进入 Park-ROADS 服务体系的第一站,定位为「轻、快、准」的入口分诊枢纽。

  • 「轻」:从访问网站到完成首次画像采集,全流程 5 分钟内完成;

  • 「快」:智能匹配引擎在 3 分钟内推荐合适的体检套餐与咨询师;

  • 「准」:基于园区类型(产业园 / 科技园 / 物流园 / 文创园 / 综合园)与客户身份(政府 / 园区 / 资本 / 学术)进行精确分诊。

3.2 核心功能

3.2.1 客户画像采集

通过分层式问卷采集客户的 5 类基本信息,作为后续所有系统的「主数据」基础。

  • 园区基本信息:类型 / 规模 / 区位 / 建筑面积 / 入驻企业数;

  • 管理主体信息:政府 / 国资 / 民营 / 混合;领导班子结构;

  • 数字化现状:已有系统、数据资产、IT 团队规模;

  • 主要痛点:招商难、运营难、双碳压力大、考核压力大、招商考核未挂钩等;

  • 决策窗口:即将立项 / 半年内 / 一年内 / 长期跟踪。

3.2.2 智能分诊

挂号系统内置一个分诊引擎,根据 G/B/A 客户类型、园区成熟度、痛点紧迫度,自动将客户路由到对应的咨询师组与服务套餐。

  • G 政府客户 → 政策与战略组(资深咨询师 2 名);

  • B 园区客户 → 园区运营组(实施工程师 + 资深顾问);

  • A 学术客户 → 研究与培训组(首席研究员 + 培训师);

  • 紧急程度高(已有立项) → 加急通道,24 小时内电话回访。

3.2.3 服务推荐与咨询师匹配

分诊后,系统给出三档推荐方案,客户可自行选择或预约咨询师确认。

  • 入门套餐:免费自助体检 + 体检报告(¥0);

  • 标准套餐:专家辅助体检 + 体检解读会(¥588 - ¥3,888);

  • 综合套餐:体检 + AI 诊断 + 专家会诊一次性打包(¥18,800 起)。

3.3 对应现有 Park-ROADS 模块

  • 首页:作为挂号入口,改造为「五系统服务概览 + 一键挂号」模式;

  • 关于:扩展为「咨询师介绍 + 团队资质 + 案例摘要」展示页,提升信任感与转化率。

第四章 · 园区数字化体检系统

4.1 功能定位

体检系统是 Park-ROADS 最重要的「标准化产品」,定位为「人人可做、一小时完成、当天出报告」的园区数字化健康标准化体检。

体检系统的设计哲学:像三甲医院的常规体检一样,所有客户使用同一套标准化指标,产出可比较、可累积、可追溯的体检数据。

4.2 核心功能

4.2.1 DTSI 基础体检包

DTSI(Digital Transformation Sliders Index)是 Park-ROADS 自研的园区数字化滑杆评分体系,经过 AHP 一致性验证(CR = 0.0283,远低于 0.1 的可接受阈值)。

  • 35 个滑杆覆盖战略 / 技术 / 数据 / 组织 / 运营 / 文化 / 生态 7 大维度;

  • 每个滑杆 0-100 评分,配套 5 级文本描述与可视化提示;

  • 7 主体雷达图(政府 / 园区 / 资本 / 学术 / 企业 / 居民 / 生态)自动生成;

  • 全流程 60 分钟内完成,移动端 / PC 端均可使用;

  • X-Flow Dynamics 引擎实时计算各滑杆之间的耦合效应。

4.2.2 自助 vs 专家辅助两种模式

  • 自助模式:客户独立填写,系统自动生成报告,适合首次了解 Park-ROADS 的客户;

  • 专家辅助模式:Park-ROADS 团队工程师陪同填写,过程中辅导各项指标的口径与证据采集,适合预算充足、希望一次性高质量完成的客户。

4.2.3 体检报告自动生成

体检报告分为基础版与专家版两类,分别面向不同付费档位。

  • 基础版(¥0):雷达图 + 35 滑杆总览 + 一句话健康总结,共 5 页;

  • 专家版(¥3,888):基础版 + 同行业 Top10 类比 + 3 条改善路径 + 投入产出测算,共 25-35 页。

4.2.4 多人协作体检

园区是一个多主体协作系统,单一视角的体检容易失真。Park-ROADS 体检系统支持 7 主体代表分别填写问卷,系统自动汇总并标记「主体间共识差距」。

  • 政府代表(管委会 / 街道办)填写 7 个治理维度滑杆;

  • 园区代表(运营公司 / 物业)填写 9 个运营维度滑杆;

  • 资本代表(投资人 / REITs)填写 4 个资本回报维度滑杆;

  • 学术代表(高校 / 智库)填写 3 个研究合作维度滑杆;

  • 企业 / 居民 / 生态代表分别填写其相关滑杆;

  • 系统输出「共识雷达图 vs 差距雷达图」,识别园区内部认知错位。

4.3 体检 → 诊断 → 服务闭环

体检报告不是终点,而是下一阶段服务的入口。报告底部默认显示三类升级建议:

  1. AI 诊断升级(¥988):基于体检结果,自动给出 Top 5 改善杠杆与失衡告警;

  2. 专家会诊升级(¥18,800):3 名资深顾问会诊 2 小时,深入分析 3 个最严重问题;

  3. 综合服务升级(¥58,800+):一次性打包 AI + 专家 + 多角色会诊,适合即将立项的客户。

4.4 对应现有 Park-ROADS 模块

  • 政策沙盘:作为体检的 What-if 引擎,允许用户在体检结果上拖动滑杆,模拟未来 12 / 24 / 36 个月的发展轨迹;

  • 调查问卷:作为多人协作体检的工具,扩展为 7 主体专属问卷模板。

第五章 · 园区数字化诊断系统

5.1 功能定位

诊断系统是 Park-ROADS 的「核心价值高地」。如果体检是「测温量血压」,诊断就是「找出病因、确定病灶、给出处方」。

诊断系统融合 AI 自动诊断、资深咨询师专家会诊、多角色多视角会诊三种诊断方式,产出可执行、可量化、可追溯的诊断报告。

5.2 核心功能

5.2.1 AI 自动诊断

基于体检数据,AI 引擎自动生成 5 张洞察卡片,使客户在不超过 60 秒内获取核心结论。

  1. 最弱主体识别:7 主体中得分最低的主体及其结构性原因;

  2. Top 5 杠杆:基于 X-Flow Dynamics 影响传播分析,给出投入产出比最高的 5 个改善杠杆;

  3. 失衡告警:7 主体之间的认知差、资源差、能力差 → 自动标记 3-5 个高危失衡点;

  4. 类比园区:基于行业库,找到 3 个与本园区高度相似但发展更快的类比园区;

  5. What-if 模拟:用户可在 AI 诊断卡片中拖动 5 个关键滑杆,实时查看未来 24 个月 DTSI 变化轨迹。

5.2.2 专家深度访谈

AI 诊断的局限在于无法捕捉「上下文知识」与「政治性约束」。专家访谈通过 10 位资深咨询师轮值,提供 AI 之外的诊断深度。

  • 每位资深咨询师覆盖一个细分领域:战略、招商、运营、双碳、人才、资本、政策、生态、文化、技术;

  • 访谈采用结构化模板,1-2 小时完成;

  • 访谈输出三类内容:隐性问题揭示、政治性约束识别、可行性边界界定。

5.2.3 多角色会诊

园区数字化转型涉及多元利益相关者,任何单一视角的诊断都会失真。Park-ROADS 提供 4 视角会诊机制。

  • 政府视角:政策契合度、考核口径、补贴申请可行性;

  • 园区运营视角:成本投入、运营效率、租户体验;

  • 资本视角:回报周期、估值提升、退出路径;

  • 学术视角:方法论严谨度、可复制性、行业贡献度。

4 视角分别独立评估,然后由首席诊断顾问主持「四方会诊会议」,形成共识诊断结论。

5.2.4 诊断报告

诊断报告分为在线交互版与 PDF 归档版两种格式,均包含以下 6 部分。

  1. 执行摘要:1 页核心结论,适合给一把手汇报;

  2. 现状画像:体检数据回顾 + 7 主体雷达图 + 同业类比;

  3. Top 5 问题深度剖析:每个问题配根因分析 + 影响范围 + 改善紧迫度;

  4. 改善路径建议:3-5 条候选路径,每条配工期、投入、产出预测;

  5. 投入产出测算:基于 X-Flow Dynamics 引擎,给出 12 / 24 / 36 个月的预期回报;

  6. 下一步服务推荐:对接服务系统,给出 3 档服务套餐推荐。

5.2.5 Second Opinion 第二意见

Park-ROADS 提供「第二意见」服务,允许客户拿着已有诊断报告(无论是 Park-ROADS 自身还是其他咨询机构)申请独立复评。

  • 由首席诊断顾问之外的资深顾问独立完成;

  • 48 小时内出具复评意见;

  • 当结论分歧时,触发「三方会诊」机制。

10.2 SaaS = 挂号 + 体检(高频低客单)

E.3 看板可视化设计原则

  • 每个 KPI 配「当前值 + 目标值 + 趋势图(过去 12 周)」;

  • 超出 / 低于目标 ±10% → 黄色预警,±20% → 红色预警;

  • 漏斗层用 Sankey 图,展示从挂号到评估的客户流动;

  • 财务层用瀑布图,展示收入构成与毛利贡献;

  • 团队层用蜂窝图,每个团队成员的人均产出可点击钻取。

E.4 看板访问权限

  • 首席诊断官 / CEO / COO:全权限;

  • 五系统总监:本系统全权限 + 其他系统只读;

  • 合伙人:对应系统全权限 + 整体只读;

  • 投资人:整体只读(按月汇总);

  • 一线团队:本系统只读 + 个人 KPI 可读写。

E.5 KPI 复盘会议节奏

  • 每日:挂号系统站会(15 分钟,客户成功团队);

  • 每周:五系统 KPI 周会(60 分钟,五系统总监 + 首席诊断官);

  • 每月:跨系统月度复盘(2 小时,五系统总监 + 合伙人);

  • 每季度:战略复盘(半天,合伙人 + 外部顾问);

  • 每年:战略 + 产品 + 客户三层年度大盘点(2 天 + 投资人参与)。
